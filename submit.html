Recommended workflow

This static starter site is designed to connect to a GitHub Issue Form. The issue triggers a GitHub Action, which drafts a candidate entry. You review the pull request before it appears in the verified catalogue.

  1. Create the GitHub repository as joshigunjan/EO-FM-Living-Atlas, then enable GitHub Pages.
  2. Enable GitHub Pages for the repository.
  3. Use the included issue form at .github/ISSUE_TEMPLATE/add-paper.yml.
  4. Use the included workflow at .github/workflows/extract-candidate.yml as the automation starting point.

Why review first?

New entries should not be automatically published as verified. The automation should create a candidate row first, because source papers often use different meanings of “foundation model,” “open source,” “benchmark,” and “pretraining scale.”
