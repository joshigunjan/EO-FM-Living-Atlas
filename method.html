Method

How entries are organized.

Scope

The atlas focuses on foundation-model research for Earth observation and remote sensing. It includes visual, multimodal, SAR/radar, spectral and hyperspectral, Earth-system, generative, vision-language, MLLM, and benchmark-oriented works when they contribute to the EO foundation-model ecosystem.

What each row records

Each catalogue entry is organized around scientific and practical comparison fields: model scope, input modalities, architecture family, downstream tasks, openness, and direct source links. The goal is to make it easy to compare what a model actually supports rather than only listing its name and paper title.

Link policy

The catalogue prioritizes direct links to the original paper, official code repository, official project page, and model weights. External registries can be useful for cross-checking, but the table itself should point readers to the primary sources.

Downstream-task policy

Downstream tasks are recorded as specifically as possible. For example, instead of writing only “segmentation,” the catalogue should indicate whether the paper reports land-cover segmentation, flood mapping, crop-type mapping, building/change detection, burn-scar mapping, visual question answering, captioning, retrieval, regression, or other named tasks.

Minimum fields for a complete row

  • Model or benchmark name
  • Original paper link
  • Model scope and category
  • Input modalities and sensors where available
  • Architecture family
  • Named downstream tasks
  • Code, project, and weight links where available
  • Openness and caveats
